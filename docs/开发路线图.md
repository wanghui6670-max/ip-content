## P1：网页端产品化

目标：让 `web-dashboard` 成为稳定的内部内容中台入口，而不是一次性评审页。

### 核心方向

- [ ] 拆分 `main.js`，把命令生成、模式提示、面板渲染逐步模块化
- [ ] 增加更明确的数据模式提示（fallback / full source）
- [ ] 增加数据更新时间、来源上下文和可信度说明
- [ ] 增加筛选 / 搜索 / 排序的交互细节完善
- [ ] 增加任务视图 / 发布视图 / 复盘视图的独立入口
- [ ] 引入更明确的空态提示和状态色

## P2：同步能力独立化

目标：不再依赖人工导出或评审包回退构建，建立正式的同步层。

- [ ] 建立 `packages/feishu-sync`
- [ ] 明确镜像清洗规则
- [ ] 支持镜像增量更新
- [ ] 增加同步日志与失败告警
- [ ] 形成统一的镜像输入契约

## P3：领域模型与 CLI

目标：把内容中台核心规则代码化，并从 UI 脚本中抽离出来。

- [ ] 建立 `packages/content-domain`
- [ ] 定义选题、任务、回执、复盘实体
- [ ] 抽象状态流转规则
- [ ] 建立 `tools/ip-cli`
- [ ] 支持本地导出、校验、修复、预览

## P4：上线与协作治理

目标：形成可持续迭代的私有内部工具，而不是单点可运行页面。

- [ ] Vercel 私有部署 / 访问保护
- [ ] Cloudflare Access / Vercel Auth
- [ ] GitHub Required Checks
- [ ] 发布说明 / 变更记录
- [ ] 数据暴露边界审计

## 当前执行顺序

推荐按这个顺序推进：

1. P0 工程护栏与仓库收口
2. P1 `web-dashboard` 小步重构
3. P3 `content-domain` 最小骨架
4. P2 `feishu-sync` 基础实现
5. P3 `ip-cli` 与 P4 协作治理

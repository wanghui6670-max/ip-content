# 仓库初始化说明

本仓库的目标不是“把评审包原样塞进 Git”，而是把 `ip-content` 升级成一个可以长期开发的正式工程。

## 为什么必须新开仓库

- 当前 `light-meal` 仓库与 `ip-content` 不是一个业务边界。
- 内容中台会继续扩展同步、发布、分析、CLI 与自动化能力。
- 后续会形成独立的 Issue、版本节奏、部署配置和权限模型。

## 推荐边界

### 放进新仓库

- `apps/web-dashboard`
- `data/feishu-mirror` 的可提交镜像
- `docs/` 里的 PRD、架构说明、上线说明
- 后续的 `packages/feishu-sync`
- 后续的 `tools/ip-cli`

### 不要继续放在轻食项目里

- `ip-content` 网页端主应用
- 飞书镜像与 schema
- 内容系统级 PRD / 架构文档
- 中台状态机和自动化脚本

## 第一个里程碑

先把仓库变成“能独立 build、能独立部署、能独立收敛问题”的形态；不要一上来就追求全自动发布和全链路同步。

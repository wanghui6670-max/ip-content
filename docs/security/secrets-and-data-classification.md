# Secrets and Data Classification

## 数据分级

### L0：公开可见

可以公开提交和公开部署的内容：

- 通用代码
- 通用 UI 组件
- 示例配置
- 不含业务上下文的 schema

### L1：内部工程信息

建议只在私有仓库 / 受控部署中出现：

- 目录结构说明
- 任务流转规则
- CLI 命令约定
- 构建与部署流程

### L2：内部运营数据

不应在公开仓库或公开部署中出现：

- 选题排期
- 协作者安排
- 发布回执明细
- 复盘结论
- 内容资产映射关系

### L3：敏感凭证与访问入口

绝不能提交：

- token
- base-config.json 真值
- 私钥 / 证书
- 可直接访问内网系统的链接

## 当前仓库的特别注意事项

`apps/web-dashboard/src/generated/dashboard-data.json` 当前仍被用于 fallback 构建模式。

这意味着：

- 它可以帮助仓库在缺失完整源数据时继续构建
- 但它也可能包含内部运营上下文
- 所以仓库与部署的访问边界必须谨慎控制

## 提交前检查

提交前请至少检查：

1. 是否新增了真实配置文件
2. 是否把内部 URL 带进了前端产物
3. 是否把内部计划/排期/协作者信息意外提交
4. 是否让 fallback 数据暴露范围扩大

## 推荐实践

- 一律使用 `base-config.example.json` 作为示例
- 真实配置只放本地或受控 secret store
- 生产部署默认隐藏飞书主库 URL
- 对涉及镜像数据的改动，优先先审查生成产物再合并
